# 좌석배치 설계 (P3 확장 · 3/4)

## 배경
합의된 순서(통계 대시보드→수행원 관리→좌석배치→동선) 중 통계 대시보드·수행원 관리는 완료·배포됨. 이번은 세 번째 항목인 좌석배치. 사용자 요구사항(그대로 인용): "행사장 좌석 구조가 다 다를거니깐 의전매뉴얼을 기준으로 해서 좌석배치도를 가로 O칸 세로 O칸 그리고 복도 구역 지정해서 나누고 의전서열에 따라 추천배치를 한 이후에 임의로 변경배치 할 수 있도록", "행사 생성 시 정해두고 바꿀 수 있게", "무대 위치는 항상 표기".

## 목표
행사마다 다른 좌석 구조를 가로×세로 그리드로 표현하고, 복도(통로) 구역으로 블록을 나눈 뒤, 기존 의전서열 계산을 그대로 재사용해 자동 추천 배치 → 수동 재배치까지 지원한다.

## 데이터 모델 (새 컬렉션 없음)
- `events`에 `seatConfig:{cols:number, rows:number, aisleRows:number[], aisleCols:number[]}` 추가. `aisleRows`/`aisleCols`는 "그 행/열 전체가 복도"라는 뜻(칸 단위가 아니라 행·열 단위 — 실제 좌석배치 관례와 일치하고 구현이 단순함). `cols`/`rows`는 행사 생성 모달에서 입력, 이후에도 좌석배치 화면에서 다시 바꿀 수 있음.
- `eventGuests`에 `seat:{row:number,col:number}|null` 추가.

## 알고리즘 (순수 함수, DOM 비의존)
1. `seatPriorityCoords(cols,rows,aisleRows,aisleCols)` — 무대(row 0 쪽)에서 가까운 줄부터, 각 줄 안에서는 중앙 열 → 바깥쪽으로 번갈아가며 우선순위를 매긴 좌표 배열을 반환. `aisleRows`/`aisleCols`에 해당하는 좌표는 제외.
2. `autoAssignSeats(evId)` — 기존 `computeOrder(evId)`(의전서열, 이미 구현됨)가 정렬한 순서대로 `seatPriorityCoords(...)` 좌표에 내빈을 채워 `eg.seat`를 설정. 기존 "자동 추천 적용"과 동일하게, 다시 실행하면 전체를 재계산해 덮어씀.
3. `swapSeats(evId, egIdA, egIdB)` — 두 내빈의 `seat`를 서로 바꿈(한쪽이 빈 좌석 대상이면 이동으로 처리).

## 화면
- 신규 페이지 "좌석배치"(준비·편집 섹션, 의전순서 옆). 권한은 `order`와 동일(admin/chief/manager 편집, 그 외 조회 또는 미노출).
- 그리드 상단에 **"무대"** 배너를 항상 고정 표시(그리드는 무대 쪽이 위).
- 그리드 렌더링: `cols×rows` 칸, `aisleRows`/`aisleCols`에 해당하는 행/열은 통로로 표시(빈 회색 띠), 나머지 칸은 좌석(배정된 내빈 이름 표시 또는 빈 좌석).
- 상단 컨트롤: "추천배치 적용" 버튼. (가로/세로 칸수는 행사 생성/수정 모달에서 입력 — 이 페이지에서는 표시만 하고 재입력은 행사 수정 모달로 유도)
- 수동 재배치: 기존 의전순서 페이지가 이미 쓰는 네이티브 HTML5 드래그앤드롭 패턴을 그대로 재사용 — 배정된 좌석을 드래그해 다른 칸에 놓으면 서로 자리 교체(빈 칸이면 이동).

## 이벤트 생성/수정 모달 변경
`openEventModal`에 "가로 칸수"/"세로 칸수" 입력 2개 추가(기본값 10×8). `saveEvent`에서 `ev.seatConfig`에 저장(기존 `aisleRows`/`aisleCols`는 보존, 신규 행사는 빈 배열로 시작).

## 에러 처리
- 좌석 수(복도 제외)보다 내빈 수가 많으면, 넘치는 인원은 배정하지 않고 "배치 안 됨 N명" 안내 표시(에러로 막지 않음).
- 그리드 크기가 아직 설정되지 않은 행사는 좌석배치 페이지에서 "행사 수정에서 가로·세로 칸수를 먼저 설정하세요" 안내.

## 테스트
`tests/run.js`에 `seatPriorityCoords`(무대에서 가까운 순·중앙 우선 순서, 복도 제외), `autoAssignSeats`(의전서열 순서대로 채워지는지, 좌석 부족 시 넘치는 인원 처리), `swapSeats`(교체/이동) 단위테스트 추가.

## 범위 밖
- 좌석표 인쇄/내보내기 없음(화면 내 배치만).
- 원형/비정형 좌석 배치 없음(직사각형 그리드만).
