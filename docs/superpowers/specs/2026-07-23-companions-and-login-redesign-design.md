# 수행원 관리 + 로그인 화면 개선 설계

## 배경
P3 확장 순서(통계 대시보드→수행원 관리→좌석배치→동선) 중 통계 대시보드는 완료·배포됨. 이번 스펙은 두 번째 항목인 **수행원 관리**와, 사용자가 추가로 요청한 **로그인 화면 시각 개선**을 함께 다룬다. 사용자가 설계를 승인하고 이후 단계는 확인 없이 끝까지 진행하도록 요청함 — 구현 중 막히는 사안이 없는 한 중간 승인 질문 없이 계획→구현→리뷰→배포까지 진행한다.

## 1. 수행원 관리

### 목표
VVIP/VIP 내빈에 동행하는 비서·운전기사·경호원 등의 명단(이름/역할/연락처)을 행사별로 등록해, 영접 담당자가 한눈에 볼 수 있게 한다.

### 데이터 모델
새 컬렉션 없음. `eventGuests`에 필드 추가: `companions:[{name,role,phone}]` (기본 빈 배열). `role`은 자유 텍스트(비서/운전기사/경호원 등 고정 목록 강제 안 함).

### 로직 (순수 함수, DOM 비의존)
- `addCompanion(list)` — `{name:'',role:'',phone:''}`를 추가한 새 배열 반환
- `removeCompanion(list,idx)` — 해당 인덱스를 제거한 새 배열 반환
- `saveEg()`에서 저장 시 이름이 빈 행은 제외하고 trim 처리

### UI
- **입력**: 기존 "내빈 행사 설정" 모달(`openEgModal`)에 "수행원" 섹션 추가. 임시 배열 `EG_COMP`에 담아두고 이름/역할/연락처 입력 행 + `+ 수행원 추가`/삭제 버튼으로 조작, "저장" 시에만 `eg.companions`에 반영.
- **표시**: VVIP 영접 관리 페이지(`renderReception`) 카드에 수행원이 있으면 "수행원 N명" 요약 표시.
- 결과보고·통계에는 노출하지 않음(운영 정보이지 보고 대상 아님 — 범위 최소화).

### 권한·감사
별도 권한 불필요(기존 `guest.edit`/`assign` 권한 범위 내). `saveEg()`의 기존 `audit()` 호출에 수행원 수 변화를 포함시켜 기록.

### 테스트
`tests/run.js`에 `addCompanion`/`removeCompanion` 순수 함수 단위테스트, 저장 시 빈 이름 제외 로직 테스트 추가.

## 2. 로그인 화면 개선

### 배경
현재 로그인 화면(`renderLogin()`)은 진한 녹색 그라디언트 배경(`.login-bg`) 위에 흰색 카드(`.login-card`)를 올린 구조다. 사용자가 첨부한 참고 이미지는 카드 자체도 어두운 톤으로 통일하고, 우상단 "화성특례시" 배지, 도시 스카이라인 일러스트, 역할 카드마다 색상 원형 아이콘 + 화살표, 금색 "역할 선택" 구분선을 포함한 히어로형 레이아웃이다.

### 설계 결정
이 개선은 **로그인 화면(인증 전 진입 화면)에 한정**한다. 로그인 후 내부 화면은 `DESIGN-notion.md`의 기존 디자인 시스템(따뜻한 종이 캔버스·단일 녹색 액센트)을 그대로 유지 — 진입 화면과 업무 화면의 톤을 분리하는 것은 이 요청의 범위이지, 앱 전체 디자인 시스템을 바꾸는 것이 아니다.

### 변경 내용 (CSS/HTML만, 로직 변경 없음)
- `.login-card`를 반투명 다크 패널(`rgba(6,40,25,.55)` 배경 + `backdrop-filter:blur`)로 변경, 텍스트는 밝은 색으로 전환
- 헤더에 우상단 "화성특례시 · HWASEONG SPECIAL CITY" 배지 추가
- "SPC ONE" 워드마크 아래 영문 서브타이틀 "Smart Protocol & Courtesy" 추가
- 순수 SVG로 도시 스카이라인 실루엣(건물·다리·빛나는 점) 장식 일러스트 추가 — 외부 이미지·CDN 없이 인라인 SVG
- `.login-div`("역할 선택" 구분선)를 금색 톤으로 변경
- `.demo-role` 카드에 역할별 색상 원형 아이콘 + 우측 화살표(`>`) 추가. 아이콘은 `ROLES` 객체에 필드 추가해 기존 내비게이션 아이콘 경로를 재사용(신규 아이콘 최소화)
- 하단 안내문에 자물쇠 아이콘 추가

### 테스트
`renderLogin()`이 예외 없이 문자열을 반환하는지 스모크 테스트 추가(기존 "전 역할 × 전 페이지 렌더" 패턴과 동일한 취지).

## 에러 처리
- 수행원 이름 없이 저장 시 해당 행은 조용히 제외(에러 없음, 사용자가 실수로 빈 행을 남겨도 저장 실패하지 않음).
- 로그인 화면은 순수 프레젠테이션이라 별도 에러 케이스 없음.

## 범위 밖
- 수행원 개별 체크인/도착 추적 없음(명단 관리만).
- 로그인 화면 외 내부 페이지 디자인 변경 없음.
