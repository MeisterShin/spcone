# 통계 대시보드 설계 (P3 확장 · 1/4)

## 배경
CLAUDE.md 우선순위표의 P3(확장) 항목 중 좌석배치·동선·수행원 관리·통계·조직 인증 5가지가 미구현 상태였다. 사용자가 전부 추가를 요청했으나 각각 독립된 하위 시스템이라 순서대로 스펙→구현을 진행하기로 했다. 합의된 순서: **통계 대시보드 → 수행원 관리 → 좌석배치 → 동선**. 이 문서는 첫 번째인 통계 대시보드만 다룬다.

## 목표
- 새 데이터 모델·Firestore 스키마 변경 없이, 기존 `events`/`eventGuests`/`checkinLogs` 컬렉션만으로 통계를 계산한다.
- 담당자가 "이번 행사가 잘 진행되고 있는가"와 "우리 조직의 의전 운영이 개선되고 있는가"를 한 화면에서 볼 수 있게 한다.

## 범위
- **행사 상세 탭**: 현재 선택된 1개 행사의 참석률·도착 분포·VVIP 응대시간·지연/취소 비율
- **행사간 추이 탭**: 여러 행사를 날짜순으로 비교하는 참석률·평균 지연시간 추이, VVIP 응대 완료율

## 아키텍처 / 데이터 흐름
새 컬렉션·필드 없음. 전부 파생 계산.

1. `reportData(evId)`(기존, `spc.html` 결과보고 섹션에 있음)를 그대로 재사용해 참석률·도착률·10분 단위 도착 분포·평균 도착시각을 얻는다. 결과보고와 통계 대시보드가 같은 함수를 쓰므로 두 화면의 숫자가 항상 일치한다.
2. VVIP 응대시간(도착→영접완료 소요)은 새 함수 `receptionDuration(evId)`를 추가한다. `S.checkinLogs`에서 같은 `eventGuestId`에 대해 `action==='checkin'` 로그 시각과 `action==='reception_complete'` 로그 시각의 차이를 구해 평균·최댓값을 낸다. VVIP가 아닌 내빈은 제외.
3. 행사간 추이는 `S.events`를 날짜순 정렬한 뒤 각 행사 id로 `reportData()`를 호출해 배열로 모으는 새 함수 `crossEventStats()`로 만든다. 행사가 2건 미만이면 추이 대신 안내 문구를 보여준다.

## 컴포넌트
- `renderStats()` — 새 페이지 함수. `PAGES` 맵에 `stats: renderStats` 추가.
- `NAV` 배열의 `'보고 · 관리'` 섹션, `결과보고` 바로 다음에 `{id:'stats',perm:'stats',label:'통계 대시보드',...}` 추가.
- `P_ALL`에 `'stats'` 추가. `ROLE_PERMS`에서 `report`를 가진 역할(admin·chief·manager·viewer)에 `stats`도 동일하게 부여.
- 차트는 외부 라이브러리 없이 순수 SVG로 직접 그린다(결과보고 페이지의 막대그래프와 동일한 방식). 막대그래프 + 라인차트 2종만 있으면 충분.

## 화면 구성
- **행사 상세**: 스탯카드(참석률(도착/명단)·미도착/불참·VVIP 도착·VVIP 응대완료·평균 응대시간·최대 응대시간) → 10분 단위 도착 분포 막대그래프(결과보고와 동일 시각화 재사용)
- **행사간 추이**: 참석률 추이 라인차트, 행사별 VVIP 응대완료율 막대그래프. 표본이 2건 미만이면 "추이를 보려면 행사가 2건 이상 필요합니다" 안내로 대체.
  (지연 수·취소 수 스탯카드와 평균 지연시간 추이 라인차트는 기존 데이터에서 지연 메트릭을 새로 정의하지 않고도 계산 가능한 범위로 구현 시 축소되어 제외됨)

## 에러 처리
- 선택된 행사가 없으면 기존 페이지들과 동일하게 `emptyState('행사를 선택하세요')` 표시.
- 체크인 로그가 없어 응대시간을 계산할 표본이 0건이면 "데이터 없음"으로 표시(0이나 NaN을 그대로 보여주지 않음).

## 테스트
`tests/run.js`에 아래 케이스 추가:
- `receptionDuration()`가 도착·영접완료 로그 쌍에서 올바른 평균을 계산하는지
- 응대완료 로그가 없는 VVIP는 평균 계산에서 제외되는지
- `crossEventStats()`가 행사를 날짜순으로 정렬해 반환하는지
- 행사 1건뿐일 때 추이 UI가 안내 문구로 대체되는지 (렌더 함수 스모크 테스트)
